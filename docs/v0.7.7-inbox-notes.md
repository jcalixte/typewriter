Two commands for a Zettelkasten `_inbox/` habit: one to capture into today's note,
one to open the oldest note for triage.

- **`:inbox`** (alias `:in`) ‚Äî open today's fleeting note, `_inbox/YYYY-MM-DD.md`
  under [`REPO_DIR`], creating it prefilled with a `# DD/MM/YYYY` heading if it is
  new. If today's note already exists it is *opened*, never overwritten.
- **`:oldest`** (alias `:old`) ‚Äî open the oldest note in `_inbox/` for cleanup.

## Why

The writer keeps fleeting notes in a git-tracked `_inbox/` folder, one file per
day named `YYYY-MM-DD.md` with a `# DD/MM/YYYY` heading. Doing that by hand on the
device means typing today's date into both the filename (via `:enew`) and the
first line ‚Äî friction on a capture action that should be a single command. And
processing the backlog means remembering which note is oldest. These two commands
remove both chores.

- **Open-don't-clobber.** `:inbox` resolves today's path, then checks ‚Äî in order ‚Äî
  whether it is the active buffer, a parked buffer, or in the palette file list
  (the boot walk of the card, so "in the list" ‚áî "on disk"). Any hit *switches* to
  it; only a genuine miss creates. Re-running `:inbox` at 3pm to add to the note
  you started at 9am must never wipe it.
- **Prefilled + dirty on create.** A new note is seeded with `# DD/MM/YYYY` and a
  blank line, marked dirty so an eviction / `:w` / idle-save persists it (mirroring
  `:enew`). It lands in **Normal** with the caret on that blank line ‚Äî consistent
  with every other file open in this editor; press `i`/`o` to write. No card IO
  happens until the buffer is actually saved.
- **The `_inbox/` dir is created on demand.** The host save path
  (`Storage::atomic_write`) now `mkdir`s the target's parent when it is missing, so
  the first `:inbox` note works even on a repo that has no `_inbox/` folder yet.
  Guarded on the parent being absent, so every fixed-path save under `/sd` (and the
  common case where the dir exists) skips it.
- **`:oldest` is free.** The palette file list is sorted by path, and the notes are
  ISO-dated (`YYYY-MM-DD`), so the first entry under `_inbox/` *is* the
  chronologically oldest ‚Äî no dates parsed, no `stat` calls. It filters to
  `_inbox/‚Ä¶‚Äã.md` (skips a `_inboxes/` lookalike dir and non-markdown files). Needs
  no clock, so it works offline any time.

## The clock, and the timezone

The pure editor core has **no clock** ‚Äî the date is fed in by the host each key
batch via `Editor::set_today(Option<Date>)`, exactly as the file list and prefs
are fed. Two constraints shaped this:

- **The clock may be unset.** The editor boot path never runs Wi-Fi/SNTP (writing
  is offline-first), and the ESP32-S3 has no battery-backed RTC, so on a cold boot
  the wall clock sits at the epoch until the first `:gl`/`:gp` sync of this power
  cycle sets it. The host reports `None` while the year is `< 2020`, and `:inbox`
  refuses with `clock not set - :gl first` rather than creating `1970-01-01.md`.
  `:oldest` is unaffected. Feeding `today` every batch (not once at boot) means a
  session that syncs mid-life, or crosses midnight, always sees the right day.
- **Timezone is a pref, not a network lookup.** NTP carries UTC only ‚Äî it has no
  zone ‚Äî so "today near midnight" needs a timezone. It lives in the git-tracked
  `.typoena.toml` as `timezone` (a **POSIX** TZ string, e.g. Paris
  `CET-1CEST,M3.5.0,M10.5.0/3`; empty = UTC), applied at boot with
  `setenv("TZ", ‚Ä¶)` + `tzset()` so `localtime_r` reads the local day. It rides the
  repo clone to every device ‚Äî no wizard step, no per-device state, no geo-IP call.
  It must be the POSIX form: ESP-IDF's newlib ships **no** zoneinfo database, so a
  bare IANA name (`Europe/Paris`) would silently stay UTC.

## Scope notes

- **No git dependency.** Both commands are pure-core buffer/file-list work plus the
  existing `Effect::Load`; they build in the light editor build too. The timezone
  `setenv`/`tzset` and the `today` clock read are host-side and also feature-free.
- **`:inbox` vs `:enew`.** `:enew <file>` creates an arbitrary empty file; `:inbox`
  is the dated, prefilled, open-if-exists special case for the daily note.
- **Not palette-cycled.** `timezone` is a free-form string, so unlike `theme` /
  `auto_sync` / `scroll_margin` the `>` palette does not rotate it ‚Äî hand-edit the
  TOML. Promoting it to onboarding is a deliberate non-goal (a fresh device is UTC
  until its repo ‚Äî which carries the pref ‚Äî is cloned anyway).
