A two-phase writing cycle: a silent **focus** block, then a full-screen **rest**
card that masks the text until you choose to continue. Distraction-free by
construction вҖ” there is **no visible countdown** at any point. The device tracks
the 25-minute block silently and imposes the break; the rest card appearing *is*
the alarm.

Surfaced as **"focus"** in the UI вҖ” the "Pomodoro Technique" name is a
registered trademark, so it stays in the docs only.

## The cycle

```
 OFF в”Җв”Җ:focusв”Җв”Җв–¶ FOCUS в”Җв”Җ25 min, at next typing pauseв”Җв”Җв–¶ REST
  в–І                в”Ӯ                                       в”Ӯ
  в”Ӯ           :focus (end)                     c (continue)в”Ӯ
  в”Ӯ                в”Ӯ                                       в”Ӯ
  в””в”Җв”Җ q / Esc в—Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ q / Esc в—Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
                       (from REST)          FOCUS в—Җв”Җв”Җ c в”Җв”Җв”Җв”Җ
```

- **FOCUS** вҖ” you edit normally; every mode still works. A monotonic timer runs,
  unseen. The side panel shows a small `focus` marker. Nothing repaints on a
  clock вҖ” the marker is static text.
- **Break due** вҖ” at 25 min the block is *due*. To avoid yanking the screen
  mid-word, the rest card drops at the **next typing pause** (or immediately if
  you are already idle). A +2 min grace cap forces it if you never pause.
- **REST** вҖ” a full-screen card masks the editor. Every key is swallowed except
  **`c`** (start the next focus block) and **`q` / `Esc`** (end the session,
  back to normal editing). Rest is **untimed** вҖ” you leave when you are ready.
  Masking the text is the point: it removes the temptation to keep working.

## The rest card

Full-screen, **themed**: a background fill with centred foreground text вҖ” a white
card in light theme, black in dark. Not a hard-coded black (a full black fill is
the harshest, slowest e-paper refresh with the most ghosting risk, for no gain).
Two lines:

```
            412 words В· 26 min

        Rest вҖ” press c to continue
```

- **minutes** вҖ” block-end minus block-start via a monotonic `Instant`, always
  available offline (wall-clock time-of-day is not вҖ” it only exists after a
  Wi-Fi SNTP sync and is not plumbed to the UI thread).
- **words** вҖ” words added to the active buffer this block, `max(0, end вҲ’ start)`.
  Best-effort: switching buffers mid-block resets the baseline, so the figure
  reflects the current buffer's contribution. It is a motivational number, not
  an audit.

Entering and leaving rest are one full refresh each (~1.9 s) вҖ” fine for a
deliberate break, not a hot path. `c` repaints the editor with a full-area
partial (~630 ms) and starts a fresh block (timer + word baseline reset).

## Why no live timer

A ticking countdown is the wrong thing on e-paper: partials are ~630 ms, a
per-second repaint would flash and wear the panel, and it would need a new
idle-repaint tick in the main loop. The design sidesteps all of it вҖ” the elapsed
check is a **single one-shot threshold** in the existing idle/pause path, not a
repaint loop. You never see a clock; the break still gets imposed. This is the
point that reconciles "no live timer" with real timeboxing.

## Design notes

- **State split.** Rest is a new `Mode::Rest`, consistent with `Mode::Palette` вҖ”
  a full-screen modal already lives as a mode. Focus is an orthogonal
  `pomodoro_on` session flag, since you keep editing in Normal/Insert/Visual
  during a block. The pure editor core owns the mode transitions, the panel
  marker, the rest-card render, and key-swallowing вҖ” all host-testable. Timing
  (monotonic `Instant`) is the one host-side piece: `main.rs` holds the
  block-start instant + start word count, computes elapsed/words each loop, and
  calls `enter_rest(words, mins)` when a due block hits a pause (the same typing
  pause `refresh_stats` already keys off). `c` / `q` in Rest emit `Effect::Focus*`
  so the host resets or clears its timer.
- **Rendering.** `draw_into` branches to a `draw_rest_card` path when
  `mode == Rest`; the panel marker is a new field in `draw_panel` gated on
  `pomodoro_on`.
- **Interactions.** Save, idle-save, and `:gp` / `:gl` are unaffected during
  FOCUS. A background sync that finishes during REST defers its snackbar until
  you continue (the panel is not visible). Nothing is persisted, so a power-pull
  mid-session just comes back OFF.
